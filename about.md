## Research and Cybersecurity

My graduate research project was a Hybrid Honeypot System — a multi-sensor threat detection platform built to capture, classify, and analyse attacker behaviour.

The system integrates Cowrie for SSH honeypot simulation, Glastopf for web-based trap deployment, and Snort IDS for network-level intrusion detection. Collected data flows into a machine learning pipeline that applies anomaly detection to classify attack types without relying on static signatures. A FastAPI dashboard surfaces the results in real time.

The engineering challenge wasn't any single component — it was making them work together reliably. Snort generates high volumes of noisy data. Cowrie and Glastopf produce logs in different formats. The ML pipeline has to process all of it quickly enough to be useful. Solving that required thinking carefully about data ingestion, normalisation, and latency at every step.

What I took from this project was less about the specific tools and more about the design discipline. Defensive systems have to be honest about what they can and can't detect. A system that generates too many false positives gets ignored. One that misses real threats is worse than nothing. Getting that balance right is an engineering problem, not just a research one.
